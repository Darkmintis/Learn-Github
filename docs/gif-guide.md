# GIF Guide — make your README and profile shine

GIFs add movement and personality to your README and profile. Here's how to use them wisely.

**Why GIFs?**
- A 3-second animated GIF catches attention better than a static image
- Shows tools, workflows, or demos in action
- Makes your profile or repo stand out

**1) Keep GIFs fast and small**
- Duration: 1–3 seconds (short and punchy)
- Size: 320×180 or 400×300 pixels max
- File size: under 5 MB (GIFs can be big!)

Tools to trim and optimize:
- `ezgif.com` — upload, crop, compress, download
- GIPHY editor — search GIPHY, click "Create", trim and edit
- `ffmpeg` (advanced) — convert video to GIF

**2) Where to find GIFs**

For beginner-friendly topic GIFs:
- **GIPHY**: search "git", "coding", "hello", "rocket" — find simple, fun GIFs
- **Tenor**: similar to GIPHY, good search
- **Make your own**: screen record yourself running Git commands and convert to GIF

**Good GIF ideas for learn-github repo**
- Search "git workflow" or "terminal commands" on GIPHY
- Search "developer" or "learning" for friendly, upbeat GIFs
- Upload a screen recording of: `git init` → `git add` → `git commit` → `git push`

**3) Embed a GIF in your README**

```markdown
![description of the GIF](https://media2.giphy.com/media/v1.Y2lkPTc5MGI3NjExMzZnNmx5bTlzbmNhbzRnY254dGdndHc0d2RmaXM4a3ZvY2Y5NTV0diZlcD12MV9pbnRlcm5hbF9naWZfYnlfaWQmY3Q9Zw/du3J3cXyzhj75IOgvA/giphy.gif)
```

Why alt text? It describes the GIF for accessibility and search engines.

Good examples:
- `![git workflow demo](URL)` — describes what the GIF shows
- `![quick setup animation](URL)` — explains the action

**4) Good GIF placements in your README**

- After the main title/intro (catch attention immediately)
- In a "Getting Started" section (show the workflow)
- In a "Demo" or "Showcase" section
- In your GitHub profile README (about yourself or your skills)

**5) Popular easy-to-use GIFs for learn-github**

Search these on GIPHY or Tenor (or use a direct link):
- "happy coding" — friendly and upbeat
- "rocket launch" — shows you're getting started
- "terminal screen" — shows developer work
- "lightbulb" — for tips and ideas
- "thumbs up" — for encouragement

**6) Example: add a GIF to your profile README**

See `profile-readme.md` for a template. Add a GIF like:

```markdown
# Hi, I'm Learning GitHub 👋

![happy coding](https://media2.giphy.com/media/v1.Y2lkPTc5MGI3NjExMzZnNmx5bTlzbmNhbzRnY254dGdndHc0d2RmaXM4a3ZvY2Y5NTV0diZlcD12MV9pbnRlcm5hbF9naWZfYnlfaWQmY3Q9Zw/du3J3cXyzhj75IOgvA/giphy.gif)

I'm learning Git, GitHub, and how to code collaboratively.
```

**Tips**
- Don't use too many GIFs — 1–2 per page is enough
- Fast GIFs load quicker and look snappier
- Always test on GitHub to ensure the link works
