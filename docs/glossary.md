# Git & GitHub Glossary — simple explanations

If you hear a term you don't understand, look it up here. No judgment—everyone was a beginner once.

**Repository (Repo)**
A folder where Git tracks all file changes. Think of it as a project with a full history.

**Commit**
A saved snapshot of your work with a message explaining what changed. Like a save point in a video game.

**Branch**
A separate line of work. You can have multiple branches (e.g., `main`, `feature/login`) working at the same time.

**Main (or Master)**
The default, primary branch. Usually the version you want to be stable and working.

**Merge**
Combine changes from one branch into another. E.g., merge `feature/login` into `main`.

**Pull Request (PR)**
A request to review and merge your branch into another branch (usually `main`). On GitHub, PRs are how teams review code together.

**Clone**
Copy a repo from GitHub to your computer so you can work on it.

**Fork**
Your own copy of someone else's repo on GitHub. You can experiment without affecting the original.

**Remote**
A server copy of your repo, usually on GitHub. `origin` is the default remote name.

**Push**
Upload your commits from your computer to GitHub (to the remote).

**Pull**
Download new commits from GitHub to your computer.

**Fetch**
Download updates from GitHub *without* automatically merging them. (Pull = fetch + merge)

**Staging**
Marking files as "ready to commit". You do this with `git add`.

**Local**
On your own computer (vs. on GitHub, which is remote).

**Head**
The current commit you're looking at. Usually points to the latest commit in your branch.

**Diff**
A summary of what changed between two versions (what lines were added, removed, or modified).

**Issue**
A discussion or task on GitHub. Issues let people report bugs or suggest features.

**Label**
A tag for issues (e.g., "bug", "good first issue", "help wanted").

**Conventional Commits**
A style for writing clear commit messages (e.g., `feat: add login`, `fix: correct typo`).

**Markdown**
A simple text format that GitHub renders as pretty HTML. Used in READMEs, issues, PRs.

**Profile README**
A special repo named after your GitHub username. GitHub displays its README on your profile page.

**GIF**
An animated image file. Great for showing demos, workflows, or adding personality to your README.

Still stuck? Ask a friend, post on a forum, or open an issue in this repo—we're all learning.
